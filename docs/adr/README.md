# Architecture and design decisions

This directory records significant product, service, and technical decisions.

## Rules

- one ADR per significant decision
- keep ADRs short
- use the existing three-digit numbering convention in filenames
- prefer updating status over rewriting history
- supersede old ADRs with new ones instead of editing them beyond recognition
- include a date
- prefer one decision per ADR, even when decisions are closely related
- include a review trigger where useful

## Status values

- Proposed
- Accepted
- Superseded
- Deprecated
